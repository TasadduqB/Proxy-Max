Optimization

          
Every strategy here runs automatically inside the proxy on each request from the CLI — no copy/paste. All stages are enabled and tuned conservatively for maximum savings without breaking Claude Code's hooks, web search, subagents or tool use.

        

        

          —
        

      


      

        

          
          

            Everything on, tuned safe. The lossless stages — prompt caching (capped at Anthropic's 4-breakpoint limit so it can never error), the SQLite response cache, and ANSI stripping — carry zero risk. The gentle lossy stages use conservative settings (wide history window, large tool-description cap, mild prose mode) so they save tokens without starving the model. Dial any back if a workload misbehaves.
          

        

      


      

        

          
          Cost-saving strategies
          
          
        


        
        

          

            

              
              Prompt caching lossless · biggest win
            

            
Injects Anthropic cache_control breakpoints on the system prompt and tool definitions (Bedrock). Repeated prefixes — which dominate every Claude Code turn — then cost ~10% of normal input. Only adds cache hints, never alters content, and respects Anthropic's 4-breakpoint limit so it can never break a request.

          

          
        


        
        

          

            

              
              Response cache lossless · SQLite
            

            
Stores each upstream response in a local SQLite database keyed by the exact request. An identical request replays the stored response verbatim (streaming included) at zero upstream cost. Lossless — it only avoids re-asking the same question.

            

              
              
            

          

          
        


        
        

          

            

              
              Tool-result filtering lossless (ANSI only)
            

            
Strips ANSI colour/escape codes from terminal output before it re-enters context each turn. Escape codes are display-only, so this is content-preserving. The two extra sub-options below are lossy — leave them off for zero-risk operation.

            

              
              
              
              
              
Truncation can cut off command output or file contents — keep at 0 unless you accept that risk.

            

          

          
        


        
        

          

            

              
              Conversation history window lossy · opt-in
            

            
Keeps a sliding window of recent turns plus the opening context, dropping the stale middle. Claude Code re-sends the full transcript each call, so this caps runaway long-session cost — but dropping turns can disrupt multi-turn reasoning and long-running subagent tasks. Off by default.

            

              

                

                

              

            

          

          
        


        
        

          

            

              
              Tool-definition compression lossy · opt-in
            

            
Trims verbose tool descriptions sent with every request, preserving each tool's name and JSON schema exactly. Descriptions guide correct tool selection (web search, Task/subagents, hooks), so trimming them can reduce tool-calling accuracy. Off by default.

            

              
              
              
            

          

          
        


        
        

          

            

              
              Prose compression lossy · opt-in
            

            
Rewrites system-prompt and message prose to drop filler words, articles and pleasantries. Highest raw reduction, but it edits the system prompt and your message text — which can weaken behavioural/tool-use guarantees and corrupt hook-injected content. Off by default; use on tolerant models only.
